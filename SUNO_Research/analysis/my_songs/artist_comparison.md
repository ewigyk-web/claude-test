# アーティスト比較 — Yuto vs 他アーティスト

## Yuto (my_songs) — 全体像

| 項目 | 傾向 |
|---|---|
| 主軸ジャンル | Modern J-Rock / Alt Rock (8系統に網羅) |
| 副ジャンル | 和ロック / Rap-Rock / Future Bass / Jersey Club / Industrial |
| BPM帯 | 132-180 (148-150が中心) |
| Key | ほぼ未指定 |
| 楽器シグネチャ | 曲ごとに変動 (固定なし) |
| ボーカル | 男性Vo中心、女性Vo/Twin/Duetも併用 |
| 歌詞言語 | 日本語+英フックが基本、多言語実験あり |
| 構造指示 | 半角[Verse][Chorus]統一 |
| ネガプロ | en-dash型を部分的に使用 (-shout, -scream, -growl) |
| **最大の特徴** | **学習・実験型 + SUNOコミュニティ自己言及** |

## Yutoの強み

1. **多ジャンル実験**: 1人で8系統 (J-Rock/和/Rap-Rock/Future Bass/Jersey/Industrial/Britpop/Punk) を試す守備範囲。
2. **他アーティスト手法の吸収**: 「他人のプロンプトをパクる」「とある巨匠の真似」と明記し、研究結果を実装する研究者姿勢。
3. **SUNOコミュニティ自己言及**: LIKE LIKE LIKE / 創造せえよ / 俺が作ったことにしたい など、SUNOユーザーの共感を狙った歌詞が当たっている。
4. **REMIX文化**: 25曲中8曲がREMIX。他者の素材を再構築する柔軟性。
5. **熱血ロックの直球**: UVERworld/SIAM SHADE/HYDE系の王道アンセムをしっかり書ける。
6. **NOTES記録の徹底**: 全曲で制作意図・反応・学んだ手法を記録する研究ログ。
7. **多言語実験**: Tesoro y Colmillo の5言語混合は他アーティストにない試み。
8. **シーン記述の活用**: 横浜みなとみらい / 六本木ヒルズ / 新宿歌舞伎町 など具体的場所をプロンプトに入れる。
9. **メタタグの研究的活用**: 歌詞末尾に「complexity level」「year」「location」などのメタ情報を埋め込み、AIの解釈を誘導。

## Yutoのクセ

- **書式の不統一**: プロンプトが曲ごとに別書式。テンプレ化できていない。
- **REMIX依存**: オリジナルとREMIXが半々。完全オリジナル路線は半分程度。
- **ジャンル分散**: 強みが特定ジャンルに固定されない (=覚えてもらいにくい可能性)。
- **長文プロンプトと極小プロンプトが共存**: 統一戦略がない。
- **影響元アーティストの参照頻度が高い**: SIAM SHADE / HYDE / UVERworld への言及が多く、自分のオリジナリティの輪郭が見えにくい。
- **ボーカル指定の使い回し**: 同じフレーズ (kobushi turns / Rolled R) を複数曲で流用。

## Yutoを位置付ける一行

> **「全部試して、他人を吸収して、SUNOコミュニティの空気を歌詞にする学習者。」**

Yutoは特定スタイルへの拘りより、**「色々試す」「他者から学ぶ」「コミュニティに刺さるテーマを歌う」**という3点が中核。10者の他アーティスト全員から少しずつ手法を取り入れ、自分のテーマに統合する**ハイブリッド戦略**。
